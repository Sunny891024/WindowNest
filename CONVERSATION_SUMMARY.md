# WindowNest 对话总结

## 项目目标

为 macOS 做一个轻量的窗口整理应用 `WindowNest`，核心交互是拖动窗口时显示浮动布局板，并通过松手完成贴靠。

## 主要功能演进

- 菜单栏应用常驻。
- 支持窗口拖动时显示布局浮窗。
- 支持 `左 / 右半屏`、`全屏`、`上 / 下半屏`。
- 新增 `居中缩小` 布局，并在弹窗里提供四种布局方式的开关。
- 调整了拖拽浮层的 tile 排布，改为 2x2 网格，避免全部横排。
- 支持多显示器。
- 支持开机启动。
- 支持英文、简体中文、繁体中文。
- 重新设计了更简洁的单色 app 图标。
- 增加了 DMG 安装包和 ZIP 发布包。
- README 已整理为中英双语产品介绍。

## 关键问题与修复

- 修复了拖网页内容、拖应用内部元素时误触发浮窗的问题。
- 修复了自有窗口内容区误触发浮窗的问题。
- 修复了部分场景下浮窗选中态不是圆角的问题。
- 修复了多屏、竖屏、副屏场景下的坐标和贴靠问题。
- 修复了休眠、唤醒、长时间闲置后监听失效的问题。
- 增加了 debug 模式用于定位监听和拖动链路问题，后续又按发布要求关闭。

## 排查结论

通过前一轮排查已经确认：

- 浮窗显示本身没有问题。
- 主要问题集中在拖动起手识别、会话清理、监听恢复和多屏坐标。
- 误触发大多来自对窗口内容区或自有窗口的识别不够严格。
- 休眠后和长时间使用后失效，属于监听链路恢复不稳定的问题。
